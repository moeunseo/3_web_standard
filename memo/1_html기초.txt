서버와 클라이언트
	클라이언트: 서버에게 요청하는 대상
	서버: 요청받은 서비스를 응답해주는 대상

웹(Web)
	요청과 응답이 일어나는 장소

웹 브라우저(Web Browser)
	사용자의 요청에 맞는 주소로 찾아가서 인터넷 컨텐츠를 검색 및 열람 후	
	사용자에게 응답하기 위한 응용 프로그램 (ex. 크롬, 엣지, IE ...)


프로토콜(protocol): 통신규약 (http vs https)
	사람끼리 소통할 때 서로 이해할 수 있는 공용어를 사용하듯이 컴퓨터끼리도 공용어 사용
	이러한 공용어를 원활하게 통신하기 위해 필요한 규약이 바로 프로토콜!
	사용자의 요청에 반드시 응답하도록 약속한다.

	http://
		클라이언트와 서버 간의 웹 페이지 등의 자원을 통신하는 규약
		텍스트로 통신하기 때문에 가로채어 본다면 누구든 내용을 볼 수 있다. (보안x)

	https:// 
		자원을 암호화해서 통신하는 규약

IP(Internet Protocol)
	네트워크 상에서 인터넷에 접속할 때 다른 컴퓨터로 구별될 수 있도록 하는 고유번호를 'IP주소'라고 한다.
	<클라이언트가 찾아갈 서버의 고유한 값, 컴퓨터가 컴퓨터를 찾을 때 쓰는 고유한 값>

도메인(Domain)
	IP주소는 기억하고 이해하기 힘들기 때문에 이를 위해 이름을 부여하는 것!(DNS)
	도메인을 통해 IP주소를 검색할 수 있다.

	IP주소로 직접 접근하면 연산이 필요한 부분이 제외되거나 프로토콜이 적용되지 않을 수 있기 때문에
	반드시 도메인을 통해 사이트에 접근하기로 한다.

웹표준
	웹에서 사용하는 기술이나 프로토콜이 일관되게 정의되고 구현되어 웹 페이지가 서로 호환성을 유지하고
	다양한 플랫폼과 브라우저에서 일관된 동작을 보이도록 하는 규약이나 규칙	

	1. HTML(HyperText Markup Language)
		웹 페이지의 구조를 정의하는 언어(뼈대를 잡아줌!^^)
	
	2. CSS (Cascading Style Sheets)
		웹 페이지의 스타일과 레이아웃을 정의하는 언어, HTML문서의 시각적 표현을 결정!
	-> 보통 퍼블리셔가 진행!

	3. JavaScript
		웹 페이지의 동적인 동작을 담당하는 스크립트 언어로, 클라이언트 측에서 실행
		-> 예를 들면, 마우스를 댔을 때 나오는 이벤트 같은 거라고 생각하기! 서버를 거치지 않음!

	-> W3C (World Wide Web Consortium)와 같은 표준화 기구에서 지속적으로 업데이트 및 발전하고 있다.
	웹 표준을 준수하는 것은 웹의 안정성과 신뢰성을 높이는데 중요한 역할을 한다.

확장 프로그램
	1. Live Server
		가상 서버를 이용해 작성한 코드를 실시간으로 확인할 수 있는 확장 프로그램

	2. HTML CSS Support / HTML to CSS autocompletion
		HTML 문서와 CSS 문서에서 작성한 속성 값, 선택자 값을 서로 연결되 문서에서
		자동완성 기능을 사용할 수 있게 지원해주는 확장 프로그램
		
	3. Auto Rename Tag
		태그의 종류를 수정할 때 자동으로 닫는 태그도 수정되도록 해주는 확장 프로그램

▷ HTML의 구성요소

▶ HTML의 주석
	<!-- 주석 내용 -->
	주의할 점: 주석 내용 또한 개발자 모드로 표시되기 때문에 주의하여 사용!

▶ 태그
	HTML 문법을 이루는 가장 작은 단위
	웹 페이지를 구성하는 요소들의 역할을 정의
	ex) <태그> 모은서 화이팅! </태그> - 내용이 있을 때 태그 문법
	    여는태그   내용      닫는태그	


	    <태그> - 내용이 없을 때 태그 문법
	이걸 통틀어 '하나의 요소'라고 부른다.

▶ 속성
	태그를 의미나 기능적으로 보충하는 역할을 하며 속성명과 속성값으로 구성(key-value 형식)
	<태그명 속성명1="속성값1" 속성명2="속성값2">
	이처럼 하나의 태그에 여러 속성을 부여하고 싶다면 공백을 기준으로 구분!

▷ HTML의 주요 특징

▶ 기본적으로, 가독성을 위해 들여쓰기와 줄바꿈 기능

▶ 블럭 요소와 인라인 요소
	블럭 요소: 태그 중 남은 공간 유무와 상관없이 자동으로 줄바꿈 되는 태그
	인라인 요소: 자동으로 줄바꿈 되지 않는 태그

▶ 부모, 자식, 형제 관계 성립
	부모: head 태그, 자식: meta, title태그
	태그의 구성 모양에 따라서 부모, 자식, 형제 관계가 성립된다.
	'줄바꿈과 들여쓰기'를 통해 부모, 자식, 형제 관계를 명확하게 파악할 수 있다.
